# 機能要件（Phase 2）

更新日: 2026-02-07
参照元: 現行 `vibescribe` 実装（`main.py`, `recorder.py`, `transcriber.py`, `settings.py`, `history.py`）

## 主要機能
1. ホットキー起動録音
2. 音声文字起こし（日本語・開発用語補正）
3. クリップボード経由の自動貼り付け
4. 録音状態UI（メニューバー/トレイ表示 + オーバーレイ + 効果音）
5. 設定管理（ホットキー、クリップボード復元、最大録音時間、Push-to-Talk）
6. 履歴保存（音声 + 文字起こしメタデータ）
7. 権限チェックとエラーハンドリング（マイク/アクセシビリティ/入力監視）

## ユーザーストーリー
1. AI エージェントと開発するエンジニアとして、話した内容を修正なしで文字化したい。なぜなら誤字脱字修正に時間を使わず、開発フローを止めたくないから。
2. AI コーディング利用者として、どのアプリでも同じショートカット操作で入力したい。なぜならツールごとに操作を変えると作業効率が落ちるから。
3. 日本語で開発指示を出すエンジニアとして、専門用語を正しい表記で変換したい。なぜなら誤変換があると AI への指示精度が下がるから。
4. 長文プロンプトを使う開発者として、タイピングせずに詳細コンテキストを短時間で入力したい。なぜなら実装品質を上げるために説明量を増やしたいから。

## MVP に含める機能の優先順位（高→低）
1. 文字起こし精度（エラー率最小化）
2. 貼り付け成功率（任意アプリへの安定入力）
3. 応答速度（録音停止から貼り付け完了まで）

## MVP KPI 目標値
- 文字起こし成功率: 98%
- 貼り付け成功率: 99%
- 応答時間: 平均6秒以内

## MVP スコープ（機能単位）
- 必須:
  - ホットキーで録音開始/停止
  - 日本語音声の高精度文字起こし
  - 開発用語の補正（誤変換抑制）
  - コピー + 貼り付けの自動実行
  - OS 権限不足時の Fail Fast / Fail Loud
- あると良い:
  - メニューバー/トレイの状態表示
  - 設定変更 UI（ホットキー等）
  - 履歴保存
